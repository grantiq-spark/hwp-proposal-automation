DOCX → HWP 변환 번들 빌더

가지고 계신 DOCX 파일을 올리면, 그 파일과 한컴 한글 변환 스크립트를 한 ZIP으로 묶어 다운로드해 드립니다. 다운받은 ZIP을 본인 Windows PC에서 풀고 run.cmd를 더블클릭하면 final.hwpx가 생성돼요. 어떤 API 키도 필요 없고, 어떤 데이터도 서버에 저장되지 않습니다.

상태 대기

          받은 ZIP을 내 PC에서 실행하는 법
          
    
            
  1. 다운로드한 hwp-runner-XXXXXX.zip 의 압축을 적당한 폴더에 풉니다.
    2. 
            
  2. 필수 설치 확인:
              
      
                
    • Microsoft Word (Office 2016+ 권장)
      • 
                
    • 한컴 한글 2020 — 한 번은 직접 실행해서 라이선스 통과시켜 두기
      • 
              
    
            
    3. 
            
  3. 선택 설치 (비교 리포트 생성을 켰을 때만):
              
      
                
    • ImageMagick — imagemagick.org
      • 
                
    • Ghostscript — ghostscript.com
      • 
              
    
            
    4. 
            
  4. 풀어진 폴더에서 run.cmd 를 더블클릭. 또는 PowerShell에서 해당 폴더로 이동 후 powershell -ExecutionPolicy Bypass -File .\run.ps1 실행.
    5. 
            
  5. 창에 === DONE 이 보이면 같은 폴더의 final.hwpx 를 한글로 열어보세요.
    6. 
            
  6. 변환이 마음에 안 들면 run.cmd 를 메모장에서 열어 -Factor 2.02 값을 1.8~2.3 사이에서 조정해 보세요.
    7. 
          

        


        

          변환이 잘 안 될 때 (트러블슈팅)
          
    
            
  • Word.Application not found: Microsoft Word 정상 설치/실행 가능 상태인지 확인. Office 365 또는 정품 Office 필요.
    • 
            
  • Cannot create HwpCtrl: 한컴 한글 2020 설치 후 한 번 직접 실행해서 라이선스 인증을 통과시키세요.
    • 
            
  • ExecutionPolicy 에러: PowerShell을 일반 사용자 권한으로 열고 Set-ExecutionPolicy -Scope CurrentUser RemoteSigned.
    • 
            
  • 한컴 한글 "스크립트 보안" 경고: 한글 → 도구 → 환경설정 → 보안 → 스크립트 보안 → "보통" 또는 "낮음".
    • 
            
  • 변환된 페이지 레이아웃이 깨짐: Factor 값을 다르게 시도. 표가 많으면 보통 1.9~2.1 사이가 좋습니다.
    • 
            
  • compare 단계 실패: 비교 리포트는 선택이라 변환 자체와는 무관. run.ps1에서 compare 호출 부분 주석 처리 가능.
